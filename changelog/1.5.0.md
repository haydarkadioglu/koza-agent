# v1.5.0 — Premium Desktop GUI Cockpit

Introduced a fully integrated, zero-port desktop GUI Cockpit for Koza Agent, utilizing pywebview to provide a local desktop app experience without running an external web server.

## 🎨 Premium Glassmorphic Desktop GUI (`ui/`)

Developed a modern design system dashboard running as a standalone desktop application window.

- **Glassmorphism Design:** Deep dark mode theme overlaid with semi-transparent frosted-glass panels (`backdrop-filter: blur(16px)`).
- **CLI Palette Alignment:** Harmonized colors with CLI branding utilizing primary `Teal`/`Cyan` and accent `Gold`/`Orange` gradients.
- **Dynamic Chat Screen:**
  - Real-time Markdown parsing (code blocks, headings, bold text, etc.).
  - Collapsible tool run cards showcasing step-by-step agent execution, arguments, and return values.
  - Active generation interruption (Interrupt / Cancel) support.
- **Drag-and-Drop Kanban Board:** Native HTML5 Drag and Drop integration to update task columns (`Todo`, `In Progress`, `Done`) by dragging cards.
- **Session Memory:** Quick listing, restoring, and deleting of saved conversation histories.

## 🔌 Serverless Python-JS API Bridge (`ui/gui_app.py`)

Established a local async API bridge for communication between the UI window and Koza core.

- **`pywebview` Integration:** Standard window manager with custom Python class exposure.
- **Dynamic Provider & Model Catalog:**
  - Automatically imports configured providers (`PROVIDERS`) and models (`PROVIDER_MODELS`) from `cli/setup_constants.py`.
  - Instantly shows custom user configurations and supports "Other / Custom..." manual model overrides.
- **Comprehensive Configuration Form:**
  - Auto-toggles API key input fields depending on the active provider's requirements.
  - Adds one-click buttons for Google OAuth login (`providers.google_oauth_provider`) and Playwright Gemini browser session establishment (`cli.setup_helpers`).
  - Implements fallback provider, models, and keys setup.
  - Exposes inputs for Twilio (SMS/WA) and Telegram bot token configurations.
  - Adds quick switches for voice mode and automated test suites.

## 🌍 Language Localization (EN/TR)

Added a localization layer allowing users to switch the interface language dynamically.

- Swapping language values immediately renders localized text nodes (`data-localize`) across the app, saving preferences in local storage. English is loaded by default.

## 🚀 CLI Subcommand Integration (`cli/typer_app.py`)

Registered the desktop app command within the Typer command tree while preserving previous CLI commands.

- Run `koza gui` to boot the desktop interface window.
- Gracefully checks for `pywebview` package and displays installation help if missing.
