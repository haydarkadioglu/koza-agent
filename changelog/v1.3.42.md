# v1.3.42

## 🔌 New Providers
- **Groq** — ultra-fast inference (llama-3.3-70b, mixtral, gemma2, deepseek-r1-distill)
- **OpenRouter** — unified gateway for 200+ models (OpenAI, Anthropic, Google, Meta, Mistral)
- Kiro provider removed (not needed)

## 🔑 API Key Validation
- Setup wizard now **tests API keys** before saving — invalid keys are rejected immediately
- Spinner animation during validation ("Testing API key…")
- Clear error messages: "invalid API key", "access denied", "connection error"
- Retry prompt on failure

## 🖥️ Windows Installer
- New `install.ps1` — one-liner install via `irm | iex`
- Adds `.venv\Scripts` to user PATH permanently (registry)
- README updated with Windows install instructions

## 📡 Channel-Aware Prompts
- Each messaging channel (Telegram, Discord, WhatsApp, CLI) gets its own system prompt context
- Agent automatically knows message limits, formatting rules, and platform constraints
- All prompts converted to English (were Turkish)

## 🛑 Service Management Fixes
- `koza quit` now kills ALL koza processes (psutil-based, cross-platform)
- `koza reset` (renamed from `koza clean`) stops all services before deleting config
- Telegram bot conflict fix: CLI no longer starts bot if daemon already running
- Background services persist on terminal close (atexit + SIGBREAK handler)
- `get_daemon_port()` fix: port=0 (services-only) no longer treated as "not running"

## 🏗️ Code Refactoring
- `cli/setup.py` split into 3 files: `setup_constants.py`, `setup_helpers.py`, `setup.py`
- Setup wizard status display fixed: shows "set" immediately after configuration
- Default provider changed from "ollama" to empty (forces user to choose)
- OpenAI models updated: gpt-5.5, gpt-5.4-mini, gpt-5-mini, gpt-4.1, gpt-4o
- Process title set to "Koza Agent" / "Koza - Services" in Task Manager
