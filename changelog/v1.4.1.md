# v1.4.1

## 🎛️ Telegram Inline Keyboard & Proactive Notifications

The Telegram bot now supports inline button interactions and scheduled proactive notifications.

### New Features

- **Inline Keyboard — Tool Confirmations** — When the agent requests tool execution, ✅ Approve / ❌ Reject buttons are displayed; auto-reject after 5-minute timeout
- **Inline Keyboard — Background Task Control** — When a background task is created, 📊 Status and 🛑 Cancel buttons are attached
- **Conditional Button Display** — Buttons appear only in confirmation or task contexts; regular messages remain clean
- **Daily Summary Notification** — Every morning at 09:00, pending tasks, today's cron jobs, and active reminders are summarized
- **Cron Completion Notifications** — When scheduled jobs finish, success/failure notifications are sent to Telegram
- **Reminder System** — Schedule future messages via the `set_reminder` tool
- **ProactiveNotifier** — Singleton component bridging APScheduler threads to the Telegram event loop via `run_coroutine_threadsafe`

### New Modules

- `bots/telegram_keyboards.py` — `InlineKeyboardManager`, `PendingConfirmation` dataclass
- `bots/telegram_notifier.py` — `ProactiveNotifier` singleton (retry, queue, daily summary, cron hook, reminder)
- `skills/reminder.py` — `set_reminder` tool definition and handler

### Modified Modules

- `bots/telegram.py` — `CallbackQueryHandler` routing, `_make_permission_callback`, notifier initialization via `post_init`, inline keyboard on task creation messages
- `skills/cron_scheduler.py` — `_notify_completion` hook added
- `tools/registry.py` — `set_reminder` tool registered
