# v1.4.2

## 🎨 CLI Visual Polish & Stability Fixes

Major terminal UI overhaul with syntax highlighting, enriched status bar, and critical input/session fixes.

### New Features

- **Pygments Syntax Highlighting** — Fenced code blocks with language tags are now syntax-highlighted using Pygments Terminal256 formatter; untagged blocks fall back to cyan
- **Enriched Status Bar** — Now shows background task count, shortened CWD, and memory usage alongside model/tokens/time
- **Context-Aware Response Boxes** — Agent responses use colored borders: teal (normal), red (error), green (success), yellow (warning)
- **Progress Bar Widget** — Animated Unicode block progress indicator (░▒▓█) for long-running operations
- **Framed Multi-line Input** — Input area now has Unicode box-drawing border and supports multi-line via Escape+Enter
- **Timestamps** — Grey HH:MM timestamps on both user messages and agent responses
- **User Message Styling** — Blue borders with "You" label, distinct from agent's teal response boxes
- **Tool Argument Truncation** — Hidden args (code, script, content) show one-line summaries instead of flooding output
- **Simplified Banner** — Clean teal-to-cyan gradient logo with version, model, and help hint (removed tool categories panel)

### Bug Fixes

- **Fixed: Enter key not working** — `multiline=True` TextArea now has explicit Enter key binding that calls `validate_and_handle()` instead of relying on prompt_toolkit's default (which inserts newline in multiline mode)
- **Fixed: Coding mode false positives** — Keyword list drastically reduced to multi-word phrases only; single words like "oluştur", "ekle", "create", "add" no longer trigger coding mode
- **Fixed: Stale working memory** — Working memory is now cleared on new session start, preventing old tool calls from polluting the LLM context
- **Fixed: Session history lost on restart** — Sessions are auto-saved on exit and the last session's messages are restored on startup

### Color Palette Fixes

- `yellow` corrected from `\033[36m` (was cyan!) to `\033[33m`
- `gold` corrected from `\033[96m` (was bright cyan!) to `\033[38;5;178m`
- Added `orange` (`\033[38;5;208m`), `italic` (`\033[3m`), `underline` (`\033[4m`)

### New Modules

- `cli/ui/_progress.py` — `ProgressBar` class with thread-safe start/stop
- `cli/ui/_status.py` — `shorten_path()` and `format_status()` for enriched status bar
