# v1.3.4 — Non-Blocking Prompt UI Polish

## 🎡 Spinner — Status Bar Animation

### New OutputSpinner system
- Spinner now writes to the **status bar** instead of the output pane — overlap issues completely resolved
- Braille character animation (⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏) rotating at 100ms intervals in the status bar
- Spinner properly stopped and restarted on each event transition
- Flow: `Waiting for response…` → `Thinking…` → `Running {tool}…` → `Streaming… N tok` → `● Idle`

### Clean event lifecycle
- `thinking` → spinner starts, status bar updates
- `tool_start` → spinner stops, tool info written to output, spinner restarts with new label
- `tool_done` → spinner stops, result summary written to output, spinner restarts with "Thinking…"
- `text` (first token) → spinner stops, response box opens
- `finalize` → spinner definitively cleaned up

## 🖥️ Layout Improvements

### Input area simplified
- Unnecessary top border removed — status bar already serves as separator
- Layout: Output (scrollable) → Status Bar (1 line) → Input (1 line)
- More compact, less screen space used

### Prompt indicator colorized
- Idle: green `●` — agent is idle
- Busy: yellow `◌` — agent is working

### Output pane auto-scroll
- Automatically scrolls to bottom when new content arrives (via `get_cursor_position`)
- Empty line added after response box (visual separation)

## 🎯 Coding Mode Integration

### `/mode coding` and `/mode off`
- Coding mode fully functional in non-blocking UI
- `CodingSession.run()` events routed to StreamRenderer
- Persona boxes rendered in output pane (╭─ 🎯 Team Lead ─, ╭─ 🔧 Backend ─, ╭─ 🎨 Frontend ─, ╭─ 🧪 Test Engineer ─)

### Automatic coding mode activation
- When keywords like "write code", "create project", "implement", "refactor" are detected, coding mode activates automatically
- When session completes (`done` event), automatically returns to normal mode

### Status bar coding mode indicator
- When coding mode is active: `🎯 Coding Mode │ ◐ Reasoning… │ deepseek-chat │ 172/64K │ 2m`

## 📋 Slash Command Improvements

### Output pane redirection
- `/help`, `/memory`, `/kanban` now use `layout.append_output()` instead of `print()`
- `/reset` clears output pane + resets renderer state + resets token counter

## 🔧 Tool Info

### Enriched tool_done output
- Result summary (first 80 chars) displayed: `✓ run_command  0.51s  → Exit code: 0`
- Multi-line results show `+N lines` indicator
