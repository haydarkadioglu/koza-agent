# v1.3.4 — Non-Blocking Prompt UI Polish

## 🎡 Spinner — Status Bar Animasyonu

### Yeni OutputSpinner sistemi
- Spinner artık output pane'e değil, **status bar**'a yazıyor — overlap/iç içe geçme sorunu tamamen çözüldü
- Braille karakter animasyonu (⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏) 100ms aralıklarla status bar'da dönüyor
- Her event geçişinde spinner düzgün durduruluyor ve yeniden başlatılıyor
- Akış: `Waiting for response…` → `Thinking…` → `Running {tool}…` → `Streaming… N tok` → `● Idle`

### Temiz event lifecycle
- `thinking` → spinner başlar, status bar güncellenir
- `tool_start` → spinner durur, tool bilgisi output'a yazılır, spinner yeni label ile başlar
- `tool_done` → spinner durur, sonuç özeti output'a yazılır, spinner "Thinking…" ile başlar
- `text` (ilk token) → spinner durur, yanıt kutusu açılır
- `finalize` → spinner kesinlikle temizlenir

## 🖥️ Layout İyileştirmeleri

### Input alanı sadeleştirildi
- Gereksiz üst çizgi (border) kaldırıldı — status bar zaten ayırıcı görevi görüyor
- Layout: Output (scrollable) → Status Bar (1 satır) → Input (1 satır)
- Daha kompakt, daha az yer kaplıyor

### Prompt göstergesi renklendirildi
- Idle: yeşil `●` — agent boşta
- Busy: sarı `◌` — agent çalışıyor

### Output pane auto-scroll
- Yeni içerik geldiğinde otomatik en alta scroll (`get_cursor_position` ile)
- Yanıt kutusu sonrası boş satır eklendi (görsel ayrım)

## 🎯 Coding Mode Entegrasyonu

### `/mode coding` ve `/mode off`
- Coding mode non-blocking UI'da tam çalışıyor
- `CodingSession.run()` event'leri StreamRenderer'a yönlendirildi
- Persona box'ları output pane'de render ediliyor (╭─ 🎯 Team Lead ─, ╭─ 🔧 Backend ─, ╭─ 🎨 Frontend ─, ╭─ 🧪 Test Engineer ─)

### Otomatik coding mode geçişi
- Mesajda "kod yaz", "proje oluştur", "implement", "refactor" gibi anahtar kelimeler algılanırsa coding mode otomatik aktifleşiyor
- Session tamamlandığında (`done` event) otomatik normal moda dönüyor

### Status bar coding mode göstergesi
- Coding mode aktifken: `🎯 Coding Mode │ ◐ Reasoning… │ deepseek-chat │ 172/64K │ 2m`

## 📋 Slash Command İyileştirmeleri

### Output pane'e yönlendirme
- `/help`, `/memory`, `/kanban` artık `print()` yerine `layout.append_output()` kullanıyor
- `/reset` output pane'i temizliyor + renderer state sıfırlıyor + token sayacı sıfırlanıyor

## 🔧 Tool Bilgisi

### Zenginleştirilmiş tool_done çıktısı
- Result özeti (ilk 80 karakter) gösteriliyor: `✓ run_command  0.51s  → Exit code: 0`
- Çok satırlı sonuçlarda `+N lines` göstergesi
