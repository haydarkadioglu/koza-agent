# v2.0.1 — 2026-05-21

## 🌐 Gemini — Playwright Browser Session

### New auth mode: `playwright`
- Google cookie auth replaced with a persistent Playwright Chromium profile
- Session saved to `~/.koza/gemini_browser/` — survives restarts
- `koza setup` → Primary Provider → **gemini cookie** now opens a browser window for Google login instead of asking for cookie values
- `koza setup` → Media → **Gemini Browser Session** shows session status (`✓ ready` / `⚠ no session`); opens browser if needed

### Media Generation updates
- Image (`generate_image`) and video (`generate_video`) now use `auth: playwright` from config
- Removed all manual cookie-paste flows (`__Secure-1PSID`, `browser_cookie3` auto-extraction UI)
- `GeminiProvider._auto_extract_cookies()` — tries `browser_cookie3` first, then falls back to reading cookies from the saved Playwright profile (headless)

### Setup wizard cleanup
- `koza setup` Media section: `Gemini cookie` option → `Gemini Browser Session` with live status indicator
- Old cookie fields (`cookie_1psid`, `cookie_1psidts`) stripped from config when switching to playwright mode
- `koza provider list` shows `browser ✓/✗` for playwright auth

### Removed
- `koza gemini-login` standalone command removed — login flow handled entirely within `koza setup`
- `cli/gemini_login.py` deleted

## 🤖 Gemini Provider — Model names updated
- Corrected model name map to match current Gemini API names (2.5-flash, 2.5-pro, 2.0-flash-lite, etc.)
- Thinking-capable models: `gemini-2.5-flash`, `gemini-2.5-pro` — `supports_thinking` only set for these
- `zai`, `minimax`, `kimi` provider thinking flags reviewed and corrected
