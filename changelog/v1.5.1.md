# v1.5.1 — UI Refinements & LM Studio Support

This update brings minor UI refinements, bug fixes, and support for the popular local LLM server LM Studio.

## 🛠 Features

- **LM Studio Provider:** Added native support for `lm_studio` local endpoints (defaults to `localhost:1234/v1`).
- **API Key Visual Feedback:** Saved API keys now visually display as `••••••••••••••••` instead of an empty text box, reducing confusion while maintaining security.
- **Dynamic Chat Headers:** Agent messages now explicitly display the current provider and model in the header (e.g., `Koza — openai/gpt-4o`).
- **Interactive Setup Wizard:** Introduced an interactive command-line setup prompt wizard (`python main.py setup`) for fresh installations.

## 🐛 Bug Fixes

- **API Key Missing Alert:** The chat interface now actively displays a warning banner if no API key is detected, instead of silently failing.
- **UI Layout Fixes:** Fixed multiple UI layout issues, including broken tab switching and horizontal scrolling caused by unclosed `div` tags in the settings panel.
- **General Settings Layout:** Moved General Settings to the top of the settings sidebar and restored the "Apply" button functionality.
