# v1.4.51 — Disable Coding Session

## Changes

- **Disable Coding Session**: CodingSession (multi-persona team coding with Team Lead, Backend Dev, Frontend Dev, Test Engineer) is now disabled by default via `coding_mode.enabled = False` feature gate
- Koza handles coding requests directly through its main Agent (`stream_chat` + tool calls) — no persona orchestration
- Users can delegate coding tasks via `spawn_subagent()` as an alternative
- All CodingSession, persona, and ErrorMemory code remains intact and re-enableable by setting `coding_mode.enabled: true` in config
- `koza coding` CLI command now shows a disabled message with alternative suggestions
- `BackgroundTaskManager.create_task()` raises `RuntimeError` when coding session is disabled
- IntentRouter and InputDispatcher respect the feature gate — coding mode is never auto-activated when disabled

## Files Modified

- `config.py` — Added `enabled: False` to `coding_mode` default config
- `router.py` — Added `coding_enabled` parameter to `IntentRouter`
- `cli/input_dispatcher.py` — Added `coding_enabled` gate to `InputDispatcher`
- `cli/coding_cmd.py` — Added feature gate check to `cmd_coding()`
- `skills/agents/background.py` — Added gate check to `BackgroundTaskManager.create_task()`
- `core.py` — Propagates `coding_enabled` flag to IntentRouter
- `cli/chat.py` — Propagates `coding_enabled` flag to InputDispatcher
