# v1.4.52 — Twitter Post, Sandbox, Platform Support, User Profile

## New Features

- **Twitter/X Post**: `twitter_post` tool added — posts tweets via tweepy with OAuth 2.0 Bearer Token (write access)
- **Sub-agent Sandbox**: Write operations (`write_file`, `delete_file`, `create_dir`) for sub-agents are now restricted to their working directory (`~/.Koza/workspace/subagents/<id>/`). Path traversal attempts are blocked. The koza source repo is always protected.
- **Cross-platform support**: System prompt now explicitly handles Windows (PowerShell) vs Linux/macOS (bash/sh) command syntax
- **User Profile & Rules**: New skill for personalization
  - `user_rule_add(rule)` — User can set permanent directives (e.g. "always respond in Turkish", "use Python 3.12")
  - `user_note_add(note)` — Koza proactively records observations about the user
  - `user_profile_list()` — View all rules and notes
  - `user_profile_delete(key)` — Remove a rule or note
  - Rules are automatically injected into the system prompt on every session start
- **Telegram message reactions**: Koza reacts with 👀 when it receives a message (acknowledging it's working), then ✅ when the response is complete

## Improvements

- **Background delegation removed from auto-routing**: Telegram bot and CLI InputDispatcher no longer auto-delegate to background tasks via LLM router. Koza handles all requests directly. Users can explicitly use `spawn_subagent()` tool when background delegation is needed.
