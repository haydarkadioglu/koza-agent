# v1.4.0

## 🚀 Background Agent Delegation

Coding görevleri artık arka planda çalışıyor — ana agent sohbete devam ederken uzun süren işler daemon thread'lerde yürütülüyor.

### Yeni Özellikler

- **BackgroundTaskManager** — `CodingSession` örneklerini daemon thread'lerde çalıştıran, thread-safe görev yöneticisi
- **EventQueue** — arka plan görevlerinden gelen olayları biriktiren, `deque(maxlen=1000)` tabanlı kuyruk
- **4 yeni LLM aracı:**
  - `start_background_task` — coding görevini arka plana delege et
  - `get_background_status` — görev durumu ve ilerleme raporu al
  - `list_background_tasks` — tüm arka plan görevlerini listele
  - `cancel_background_task` — çalışan görevi iptal et
- **Telegram otomatik delegasyon** — coding anahtar kelimeleri algılandığında görev otomatik olarak arka plana atılıyor
- **Telegram bildirimler** — görev tamamlandığında veya hata aldığında kullanıcıya mesaj gönderiliyor
- **Sistem prompt entegrasyonu** — LLM ne zaman arka plan aracı kullanacağını biliyor

### Teknik Detaylar

- Mevcut `CodingSession` sınıfı değiştirilmeden yeniden kullanılıyor
- `_background_tasks` registry'si mevcut `_subagents` sisteminden tamamen bağımsız
- Modül seviyesinde singleton — CLI ve Telegram bağlamlarından erişilebilir
- Daemon thread'ler: ana process kapandığında otomatik sonlanır
- İptal mekanizması: `session.interrupt()` üzerinden idempotent

### Property-Based Testler

11 doğruluk özelliği (correctness property) hypothesis ile test ediliyor:
- Görev ID benzersizliği ve başlangıç durumu
- EventQueue sıra koruması ve pencere davranışı
- Durum raporu alan bütünlüğü
- Liste doğruluğu
- Keyword algılama sınıflandırması
- Registry bağımsızlığı ve singleton erişilebilirliği
