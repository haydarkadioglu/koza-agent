# v1.4.0

## 🚀 Background Agent Delegation

Coding tasks now run in the background — the main agent continues chatting while long-running jobs execute in daemon threads.

### New Features

- **BackgroundTaskManager** — Thread-safe task manager that runs `CodingSession` instances in daemon threads
- **EventQueue** — Accumulates events from background tasks using a `deque(maxlen=1000)` buffer
- **4 new LLM tools:**
  - `start_background_task` — delegate a coding task to the background
  - `get_background_status` — get task status and progress report
  - `list_background_tasks` — list all background tasks
  - `cancel_background_task` — cancel a running task
- **Telegram auto-delegation** — When coding keywords are detected, tasks are automatically delegated to the background
- **Telegram notifications** — Users are notified when tasks complete or encounter errors
- **System prompt integration** — The LLM knows when to use background tools

### Technical Details

- Existing `CodingSession` class reused without modification
- `_background_tasks` registry is completely independent from the existing `_subagents` system
- Module-level singleton — accessible from both CLI and Telegram contexts
- Daemon threads: automatically terminate when the main process exits
- Cancellation mechanism: idempotent via `session.interrupt()`
